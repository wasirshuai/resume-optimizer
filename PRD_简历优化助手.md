## 一、产品概述

### 1.1 产品名称

简历优化助手

### 1.2 一句话介绍

给求职者的 AI 简历诊断工具，上传简历和目标岗位 JD，精准告诉你差距在哪、怎么改，让投简历不再靠感觉。

### 1.3 产品定位

不是"简历润色工具"，而是"简历诊断工具"。

市面上大多数 AI 简历工具解决的是"怎么把话说得更好听"，但用户真正的问题是"我不知道我差在哪"。本产品的核心价值是：**做一面精准的镜子，告诉你简历和目标岗位之间的真实差距。**

## 二、目标用户

### 2.1 用户画像

| 用户群体 | 典型特征 | 核心痛点 |
|----------|----------|----------|
| 在校应届生 | 求职经验少，不知道简历该写什么、怎么写 | 投了几十份石沉大海，不知道是简历的问题还是自己不够格 |
| 工作 3-5 年跳槽者 | 有一定经验，但不擅长包装自己的经历 | 简历写得像 JD 搬运工，投出去没回响，不知道差在哪 |
| 转行 / 跨领域求职者 | 过往经历和目标岗位不匹配 | 不知道怎么把旧经验和新岗位关联起来，简历怎么改都不对味 |

### 2.2 共性痛点

市面上的 AI 简历工具只会"润色"和"改写"，但用户真正缺的是——**我不知道我的简历和目标岗位之间差在哪。**

## 三、核心场景

### 场景一：投递前诊断

小林，22 届应届生，正在投产品经理岗。她在招聘网站看到一个很心仪的岗位，JD 写得很具体，但不确定自己的简历够不够格。她把简历和 JD 都上传到简历优化助手，10 秒后得到一份详细的差距报告——原来她缺"数据分析能力"的体现，而且项目经历写得太泛了。她照着建议改了两版，再投出去。

### 场景二：海投前批量匹配（V2 规划）

阿杰，工作 3 年想转行做运营。他同时在看十几个岗位，每个 JD 要求都不太一样。他用简历优化助手批量上传多个 JD，工具告诉他：你的简历投 A 类岗位匹配度 60%、B 类岗位 75%、C 类岗位只有 35%，并建议他优先投 B 类，同时针对 A 类补齐 XX 经历再投。

### 场景三：改完再测，迭代优化（V2 规划）

小美，工作 4 年想跳槽。她根据第一次诊断建议改了简历，但不确定改得够不够好。她把修改后的简历重新上传，再跑一次同一份 JD 的诊断，工具告诉她：匹配度从 58% 升到了 72%，还有 3 个可以继续优化的点。她改了三版，直到满意才投递。

## 四、核心功能流程

### 4.1 MVP 核心流程

```
用户上传简历（PDF 或粘贴文字）
        ↓
用户粘贴目标岗位 JD 文字
        ↓
  AI 快速解析与结构化
  ├─ 简历 → 教育背景、工作经历、项目经历、技能标签
  │   ├─ 深度技术栈提取：通读全文（含项目描述、工作职责），提取所有使用过的技术/工具/框架/模型
  │   ├─ 每个项目和岗位新增 tech_stack 字段
  │   └─ 顶层 all_technologies 字段汇总去重后的全量技术清单
  └─ JD → 岗位要求、职责描述、关键词、硬门槛
      └─ 关键词标准化：别名归并为标准名称（OpenAI/通义千问/DeepSeek/Claude 等）
        ↓
  生成四维差距诊断报告
  ├─ 硬门槛匹配（学历 / 年限 / 证书等刚性要求）
  ├─ 技能匹配度（已具备 / 部分具备 / 缺失）
  │   └─ 隐含能力推断：做了 AI 产品 = 用过 AI 工具 = 判定为"已具备"，非"部分具备"
  ├─ 关键词覆盖度（ATS 关键词命中情况）
  │   └─ 语义匹配：同义词/别名均算命中（如 JD 要求 OpenAI，简历写 GPT API → 命中）
  └─ 表达优化建议（结构/量化/逻辑）
        ↓
  用户选择改进建议，生成事实约束的可编辑简历草稿
        ↓
  使用改写稿再次诊断，同一 JD 下比较版本分数变化
```

### 4.2 完整版功能流程（V2+）

在 MVP 基础上扩展：

- 每条差距配"为什么重要 + 怎么改 + 改写示例"
- 可点击每条建议直接看到修改前后对比
- 一键生成优化后的简历草稿（用户确认后 AI 改写）
- 保存诊断记录，支持多次迭代对比（匹配度提升曲线）

## 五、AI 的作用

### 5.1 AI 在各环节的职责

| 环节 | AI 做什么 | 为什么必须用 AI |
|------|----------|----------------|
| 简历 / JD 解析 | 从非结构化文本中提取结构化信息（岗位、技能、经历、成果） | 简历格式千差万别，传统规则无法覆盖各种排版和表达方式 |
| 差距诊断 | 理解语义层面的匹配度，而不只是关键词匹配 | 同样的能力可以有 100 种说法，需要语义理解才能判断"其实是一回事" |
| 隐含能力推断 | 从项目经历中推导出隐含的技术能力（如做 AI 产品 = 用过 AI 工具） | 候选人写的是"做了什么"而非"会什么"，需要 AI 推理建立关联 |
| 改进建议生成 | 针对每条差距给出具体的、可执行的改写建议和示例 | 每个人的经历不同，建议必须个性化，无法用模板生成 |
| 简历优化改写（V2） | 在保持事实的基础上，用更专业、更有说服力的表达重写简历内容 | 需要理解业务语境、行业黑话、招聘方偏好，传统规则做不到 |
| 迭代对比（V2） | 分析简历修改前后的变化，量化提升效果 | 需要语义级别的差异分析，不是简单 diff |

### 5.2 核心设计原则

AI 不是用来"给简历化妆"的，是用来当一面精准的镜子——告诉你真实的差距在哪里。

## 六、诊断维度设计

### 6.1 MVP 阶段（3 个维度）

| 维度 | 说明 | 输出形式 |
|------|------|----------|
| **技能匹配度** | 对比简历技能与 JD 要求，标注"已具备/部分具备/缺失" | 匹配率 + 逐项清单 |
| **关键词覆盖度** | 检查简历是否包含 JD 中的核心关键词（影响 ATS 过筛） | 命中率 + 缺失关键词列表 |
| **表达优化建议** | 针对简历的结构、量化程度、逻辑性给出改进建议 | 具体建议 + 示例对比 |

### 6.2 V2 扩展维度（2 个）

| 维度 | 说明 |
|------|------|
| **硬门槛匹配** | 学历/年限/证书等刚性要求是否达标 |
| **经验深度匹配** | 项目量级、职责范围、成果数据是否达到岗位期望 |

## 七、MVP 定义

### 7.1 MVP 范围

| 类别 | 是否包含 | 说明 |
|------|----------|------|
| 简历上传（PDF） | ✅ | 支持 PDF 格式上传 |
| 简历粘贴文字 | ✅ | 也支持直接粘贴简历文字 |
| JD 粘贴文字 | ✅ | 用户手动粘贴 JD 文本 |
| 三维差距诊断报告 | ✅ | 技能匹配 + 关键词覆盖 + 表达建议 |
| 简历改写 / 生成 | ❌ | V2 阶段实现 |
| 批量 JD 匹配 | ❌ | V2 阶段实现 |
| 用户系统 / 保存记录 | ❌ | V2 阶段实现 |
| JD 链接解析 | ❌ | 法律与技术风险，暂不支持 |
| 移动端 App | ❌ | 暂不需要 |

### 7.2 MVP 验证目标

| 验证项 | 验证方式 | 通过标准 |
|--------|----------|----------|
| 需求真实性 | 用户会不会主动来用、愿不愿意填完整个流程 | 完成率 > 50% |
| 诊断质量认可度 | 用户觉得诊断准不准、有没有用 | CSAT > 7 分 |
| 回访意愿 | 用户会不会主动回来用第二次 | 回访率 > 30% |

### 7.3 数据隐私

- 不写入数据库或本地文件
- 简历、JD、诊断和改写草稿仅在当前浏览器会话内用于诊断、改写和版本对比
- 刷新或关闭页面后会话数据自动清除
- 页面明确标注会话级临时保存边界，不保存 API Key

## 八、技术方案

### 8.1 技术选型

| 层级 | 技术方案 | 说明 |
|------|----------|------|
| 前端 | Streamlit | Python 原生 Web 框架，快速搭建 |
| PDF 解析 | pdfplumber | 文本提取 |
| LLM 调用 | OpenAI 兼容 API | 支持 OpenAI / DeepSeek / 通义千问等 |
| 部署 | Streamlit Cloud / 本地运行 | 零成本部署 |

### 8.2 系统架构

```
用户界面（Streamlit）
      ↓
简历文本提取（PDF → 文本 / 直接输入文本）
      ↓
LLM 结构化解析（简历 → JSON）
      ↓
LLM 结构化解析（JD → JSON）
      ↓
LLM 差距诊断（两个 JSON → 诊断报告）
      ↓
结果展示（Streamlit 页面）
```

### 8.3 核心提示词设计

产品核心由四组提示词驱动：

1. **简历解析 Prompt**：将简历文本提取为结构化 JSON（教育、经历、技能、项目），并通读全文深度提取技术栈（含项目 tech_stack、工作经历 tech_stack、全量 all_technologies 清单），同义词归并
2. **JD 解析 Prompt**：将 JD 文本提取为结构化 JSON（要求、职责、关键词、门槛），关键词标准化归并别名
3. **差距诊断 Prompt**：对比两个 JSON，生成四维诊断报告。含隐含能力推断规则（项目经验能推导出的技能判 matched）和语义关键词匹配
4. **简历改写 Prompt**：基于事实约束生成可编辑简历草稿，不编造未经证实的信息

## 十、竞品分析

| 竞品 | 核心功能 | 差异 |
|------|----------|------|
| 超级简历 | 简历模板 + AI 润色 | 侧重模板和排版，不做差距诊断 |
| ResumeWorded | 简历评分 + 关键词检查 | 侧重英文简历，中文支持弱 |
| 各类 AI 简历生成器 | 一键生成简历 | 侧重"从无到有"，不做"差距诊断" |

**本产品差异点**：不是帮你"写"简历，而是帮你"诊断"简历——告诉你差在哪、怎么改。
