> 创建日期：2026-07-12
> 当前版本：V1.0（MVP）
> 目标：从 MVP 跑通 → 可公测 → 可作为求职作品展示

---

## 一、当前状态盘点

### 已完成

| 模块 | 状态 | 说明 |
|------|------|------|
| 产品文档 | ✅ | PRD 完整，含竞品分析、版本规划 |
| 简历输入 | ✅ | 支持 PDF 上传（含 OCR 回退）+ 文字粘贴 |
| JD 输入 | ✅ | 文字粘贴 |
| 三维诊断 | ✅ | 技能匹配 + 关键词覆盖 + 表达优化 |
| 多 API 支持 | ✅ | DeepSeek / OpenAI / 通义千问 / 智谱 GLM |
| 隐私说明 | ✅ | 用完即删，不存储 |
| API 测试按钮 | ✅ | 侧边栏可验证配置 |

### 当前短板

1. **只能本地运行** — 别人无法直接访问，无法收集真实用户反馈
2. **诊断质量未验证** — 还没有真实用户使用过，不知道诊断准不准
3. **无数据追踪** — 不知道完成率、有用率等 MVP 验证指标
4. **无用户反馈渠道** — 用户用完无法表达"有用/没用"
5. **UI 还比较粗糙** — Streamlit 默认风格，不够专业

## 二、优化路线图

### 阶段一：立即可做（1-3 天）

#### 1. 修复 API 配置问题
- 在侧边栏增加每个 API 提供商的获取教程链接
- 检测到 HTML 响应时，提示"请检查 API 地址是否正确"
- 增加常见配置错误的自检逻辑

#### 2. 增加用户反馈按钮
在诊断报告底部加两个按钮：
- 👍 "诊断有用" 
- 👎 "诊断不准"

点击后弹出一个小表单：
- 如果点"有用"：问一句"最有用的是哪个维度？"
- 如果点"不准"：问"哪里不准？技能匹配/关键词/表达建议"

> 这是你 MVP 验证的第一手数据，比任何问卷都真实。

#### 3. 增加诊断报告导出功能
- 支持一键导出为 PDF 或 Markdown
- 方便用户保存诊断结果，也方便你收集案例

#### 4. 优化诊断结果展示
- 每条建议增加"严重程度"标签（🔴 严重 / 🟡 建议 / 🟢 优秀）
- 增加总分提升建议优先级排序
- 改进移动端适配（很多人会用手机看）

---

### 阶段二：部署上线（3-5 天）

> **目标：让别人能直接用，不再只在你电脑上跑。**

#### 1. 部署到云端（推荐 Streamlit Cloud）

```
GitHub 上传代码 → Streamlit Cloud 一键部署 → 获得公开链接
```

Streamlit Cloud 对公开仓库免费，几分钟就能上线。

#### 2. 配置 API Key 的两种方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| 用户自带 Key | 零成本，隐私好 | 门槛高，很多用户没有 Key |
| 你提供 Key | 用户体验好，门槛低 | 你要承担 API 费用，有滥用风险 |

> 建议：**MVP 阶段先用"用户自带 Key"**，等验证了需求再考虑提供免费额度。或者用 DeepSeek（极低成本，100 万 token 约 1 元）。

#### 3. 埋点统计

在代码中加简单的统计：
- 访问量（PV/UV）
- 诊断完成率（进入页面 → 完成诊断的漏斗）
- 反馈按钮点击率
- 平均诊断耗时

可以用 Streamlit 的 `st.session_state` + 一个简单的 JSON 文件做本地统计，或者接入 Google Analytics。

### 阶段三：功能迭代（1-2 周）

#### 1. 补齐五维诊断

| 维度 | 当前状态 | 优先级 |
|------|----------|--------|
| 技能匹配度 | ✅ 已有 | - |
| 关键词覆盖度 | ✅ 已有 | - |
| 表达优化建议 | ✅ 已有 | - |
| 硬门槛匹配 | ❌ 待开发 | P0 — 用户最关心"我够不够格" |
| 经验深度匹配 | ❌ 待开发 | P1 — 语义判断难度大 |

硬门槛匹配相对简单（规则 + AI 判断），可以快速加。

#### 2. 增加简历改写功能（V2 核心功能）

流程：
```
用户看完诊断报告
    ↓
选择要采纳的建议（勾选）
    ↓
AI 基于选中的建议改写简历
    ↓
展示"修改前 vs 修改后"对比
    ↓
用户可以逐段编辑，导出 Word
```

> 这是"从诊断到行动"的关键一跳，用户用完诊断后最想要的就是"帮我改"。

#### 3. 增加历史记录（V2）

- 用 `st.session_state` 保存本次会话的诊断记录
- 支持"重新诊断"（改完简历后重新跑一遍）
- 展示匹配度变化：V1 → V2 → V3 的提升曲线

### 阶段四：产品打磨（2-3 周）

#### 1. 优化提示词质量

这是整个产品的核心。优化方向：

| 优化点 | 具体做法 |
|--------|----------|
| 诊断准确性 | 收集 20-30 个真实案例，对比 AI 诊断和人工判断的差距 |
| 建议可执行性 | 确保每条建议都能落地，不出现"建议增加数据分析能力"这种废话 |
| 行业适配 | 不同行业的简历标准不同（互联网 vs 金融 vs 制造业），增加行业判断 |
| 简历格式兼容 | 处理更多简历格式（表格型、多栏型、纯文字型） |

#### 2. 优化用户体验

| 改进点 | 做法 |
|--------|------|
| 加载体验 | 增加"AI 正在思考"的动画，缓解等待焦虑 |
| 结果可读性 | 用卡片式布局替代纯文字，视觉层次更清晰 |
| 移动端适配 | 确保手机上也能正常使用 |
| 错误处理 | 优雅处理各种异常（网络超时、PDF 格式错误、API 额度用完） |

#### 3. 增加分享功能

- 诊断报告可以生成一个带链接的分享页面
- 用户可以发到朋友圈/微信群
- 分享页面带你的产品名称和水印（增长获客）

## 三、作为面试作品如何打磨

这个产品是你面试 AI 产品经理的核心案例。面试官会问的问题，提前准备好答案：

### 3.1 面试必问清单

| 问题 | 你需要准备的素材 |
|------|------------------|
| 为什么做这个产品？ | 你的真实痛点 + 市场调研数据 |
| 用户是谁？怎么验证需求的？ | 用户画像 + 调研过程 + 数据 |
| 和竞品有什么不同？ | 竞品对比表 + 差异化定位 |
| AI 在里面起什么作用？ | 每个环节的 AI 职责 + 为什么必须用 AI |
| 提示词怎么设计的？ | 提示词设计思路 + 迭代过程 |
| 遇到了什么困难？ | 技术难点 + 产品决策 + 解决方案 |
| 数据结果怎么样？ | 用户量、完成率、反馈数据、迭代过程 |
| 下一步怎么规划？ | 版本路线图 + 商业化思路 |

### 3.2 作品展示建议

建议准备一个「作品说明文档」，包含：

1. **产品概述**（一句话 + 截图）
2. **PRD 文档**（已有）
3. **竞品分析**（已有）
4. **技术方案**（架构图 + 提示词设计）
5. **设计决策记录**（为什么砍到 3 维、为什么先不做改写）
6. **用户反馈数据**（上线后的真实数据）
7. **迭代记录**（V1 → V2 改了什么、为什么改）
8. **线上地址**（可访问的产品链接）

> 关键：面试官看的不只是产品本身，而是你做产品的**思考过程和决策逻辑**。

## 四、优化优先级总览

```
立即做（1-3天）
├── 修复 API 配置体验
├── 加反馈按钮
├── 加报告导出
└── 优化结果展示

本周做（3-5天）
├── 部署上线（Streamlit Cloud）
├── 埋点统计
└── 分享给 5-10 个朋友试用

下周做（1-2周）
├── 补齐五维诊断
├── 简历改写功能
├── 历史记录
└── 提示词优化

后续做（2-4周）
├── 用户体验打磨
├── 分享功能
├── 用户增长
└── 商业化测试
```

---

## 五、一句话总结

> **先让 10 个真实用户用起来，再谈功能扩展。** MVP 最大的价值不是功能多，而是验证了"差距诊断"这个需求是否成立。先部署上线、收集反馈，再根据数据决定下一步做什么。
